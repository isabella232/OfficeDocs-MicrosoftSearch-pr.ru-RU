# <a name="user-adoption-guide"></a>Руководство по внедрению среди пользователей

Эта информация предназначена для администраторов Поиск (Майкрософт), руководителей управления изменениями и владельцев бизнеса.
  
Чтобы обеспечить успешное внедрение Поиск (Майкрософт) организации, рекомендуется этот четырех этапный процесс.
  
## <a name="stage-1-stakeholders"></a>Этап 1. Заинтересованные лица

Чтобы обеспечить бесперебойное внедрение и повысить уровень Поиск (Майкрософт) пользователей, определите людей, которые могут поддерживать и управлять процессом.
  
В таблице показана иерархия команды и вклад каждого человека.

## <a name="stage-2-scenarios"></a>Этап 2. Сценарии

Для обеспечения принятия пользователями важно определить и определить сценарии для конечных пользователей. Они должны включать реальные примеры того, как люди могут использовать Поиск (Майкрософт), чтобы быстрее находить необходимую информацию, отвечать на вопросы и выполнять задачи. Чтобы оценить успешность внедрения и внедрения пользователей, важно также определить метрики, которые вы будете использовать перед началом работы.
  
### <a name="understand-what-your-users-need-and-want"></a>Понять, что нужно и чего хотят пользователи
  
Когда речь заходит о поиске информации и получении ответов, ключевое значение имеет знание потребностей и болевых точек пользователя. Для получения этих данных можно [](https://Office.com) использовать количественные методы, такие как данные из Office 365 и [SharePoint](https://products.office.com/sharepoint/collaboration) поиска, или качественные методы, например опросы и фокус-группы пользователей. Это поможет вам определить, когда, где и как Поиск (Майкрософт) может обеспечить значение.
  
Для различных ролей и групп в вашей организации рассмотрите:
  
- Существующие точки боли пользователя при поиске информации.
    
- Часто используются приложения, инструменты, сайты и другие источники информации.
    
- Что пользователи хотели бы видеть, когда речь заходит о поиске информации.
    
- Применение преимуществ Поиск (Майкрософт) к организации. Дополнительные сведения см. [в Поиск (Майкрософт) Обзор.](overview-microsoft-search.md)
    
Выберите сценарии пользователей, которые будут иметь наибольшее положительное влияние для пользователей.
  
 ### <a name="identify-success-metrics"></a>Определение показателей успешности
  
Чтобы помочь вам выполнить поставленные задачи и показать положительное влияние на организацию, необходимо определить ключевые показатели успеха перед началом работы. Сохраняйте простые и измеримые цели и регулярно оценивайте производительность, чтобы можно было быстро реагировать при необходимости.
  
1. **Управление изменениями:** Определите, как измерять осведомленность о Поиск (Майкрософт) и успехе сообщений об этом. 
    
2. **Использование:** Вы можете использовать данные из панели мониторинга Аналитика, включая ежедневных и ежемесячных активных пользователей (DAU и MAU), типы контента, которые получают наибольшее большинство представлений, и ключевые слова поиска для измерения использования с течением времени. Дополнительные сведения см. [в Поиск (Майкрософт) Аналитика отчетах панели мониторинга.](./usage-reports.md)
    
3. **Удовлетворенность пользователей:** Чтобы определить базовый уровень, используйте данные, собранные в опросах и фокус-группах пользователей. Чистое удовлетворение (NSAT) и Net Promoter Scores (NPS) также могут обеспечить полезные измерения. Независимо от используемого метода, сохраняй его последовательно, чтобы со временем можно было увидеть влияние. 
    
4. **Другие измерения:** Используйте исторические и/или текущие данные для запросов на поддержку и количества времени, затраченного на поиск. 
    
## <a name="stage-3-readiness"></a>Этап 3. Готовность

Вложение времени в подготовку к Поиск (Майкрософт) приведет к положительному результату для вашей организации. Корпорация Майкрософт предоставляет набор <a href="https://go.microsoft.com/fwlink/?linkid=2114710">для</a> принятия, содержащий шаблон электронной почты объявления, легкий визуальный контент, созданный для совместного использования на внутренних сайтах связи, а также обучающую презентацию. Существует также центр <a href="https://resources.techcommunity.microsoft.com/microsoft-search/#adoption"></a> Поиск (Майкрософт), где можно найти другие ресурсы обучения и внедрения, предназначенные для Поиск (Майкрософт) успешного внедрения.
  
### <a name="product-and-content-readiness"></a>Готовность продукта и контента
  
Администраторы непосредственно влияют на процесс поиска для конечных пользователей и определяют Поиск (Майкрософт) для организации. Сюда относится выбор типов результатов, которые должны отображаться для пользователей. Эти результаты могут включать ссылки, также известные как закладки, на документы, сайты, людей, группы, расположения, беседы и другие ресурсы на рабочем месте.
  
Дополнительные сведения см. [в Поиск (Майкрософт).](setup-microsoft-search.md)
  
Редакторы — это эксперты по теме, которые помогают управлять закладкими и Q&как в своих областях знаний. Редакторы должны создавать контент, который будет адресовать пользовательские сценарии, которые были определены ранее.
  
Дополнительные сведения см. в [материалах Plan your content.](plan-your-content.md)
  
### <a name="signing-in"></a>Вход
  
Поиск (Майкрософт) пользователи должны войти в Bing [с](https://Bing.com) помощью учетной записи или учебной записи. Чтобы сделать вход максимально бесшовным для пользователей:
  
- Чтобы [убедиться Azure Active Directory](/azure/active-directory/) правильно задайте настройки, несколько пользователей попытаются войти. Дополнительные сведения см. в [документе Test single sign-on.](test-single-sign-on.md)
    
- Настройка по умолчанию браузера для устройств с управлением компанией для Bing. Дополнительные сведения см. в [браузере Set default.](/deployedge/edge-default-browser)
    
- Развертывание расширений поиска [для пользователей Microsoft Edge](https://www.microsoft.com/windows/microsoft-edge) и Google [Chrome](https://www.google.com) с помощью групповой политики. Дополнительные сведения см. [в Поиск (Майкрософт) Обзор.](overview-microsoft-search.md)
    
### <a name="draft-communications"></a>Проект сообщений
  
Используйте реальные сценарии, чтобы рассказать пользователям о Поиск (Майкрософт) и о том, как это поможет им сэкономить время. Команда управления изменениями может помочь в создании стратегии выкатки и подготовке обеспечения и кампаний для готовности конечных пользователей.
  
Начните планирование сообщений и событий для вашей организации:
  
- Раздать баннеры, плакаты или листовки в известных местах.
    
- Демонстрации или стенды продукта в личном режиме.
    
- События в Интернете для локальных и удаленных групп.
    
- У вашего исполнительного спонсора провести собрание ратуши или отправить сообщение по электронной почте.
    
- Совместное использование документации о самопосве-Поиск (Майкрософт).
    
Все сообщения должны информировать пользователей о том, как они могут войти в [Bing,](https://Bing.com) чтобы найти результаты работы.

В зависимости от организации может потребоваться обмениваться сведениями о Поиск (Майкрософт) с группами конфиденциальности и безопасности. Дополнительные сведения см. в статье [Безопасность и конфиденциальность для Поиска (Майкрософт) в Bing](security-for-search.md).
  
Убедитесь, что ваша службы поддержки знают о возможностях и функциях Поиск (Майкрософт), а также требованиях пользователей к входу, чтобы они могли более эффективно поддерживать пользователей. Дополнительные сведения см. в [Поиск (Майкрософт) в Microsoft 365](https://support.office.com/article/Learn-about-Microsoft-Search-in-Microsoft-365-b8bf5a2c-7515-40a9-9a6a-b8ed382c86bc) и [faQs.](faqs.md)
  
### <a name="evaluate-with-pilot-users"></a>Оценка с помощью пилотных пользователей
  
Начните с небольшой группы пользователей, чтобы оценить влияние Поиск (Майкрософт) и собрать ранние отзывы. Это дает вам время для изменения контента, обновления сообщений и задания ожиданий для широкого проката. Пользователи питания поиска или ранние усыновители являются отличными кандидатами для этого.
  
Чтобы помочь найти пилотных пользователей:
  
- Партнер с бизнес-спонсором в организации.
    
- Определите группу ранних усыновителей.
    
- Определите группы, которые делают большие объемы корпоративных поисков.
    
- Определите пользователей в других географических точках, включая международные.
    
Попросите пилотных пользователей предоставить ввод с помощью параметров обратной связи в Поиск (Майкрософт) или с помощью простых опросов.
  
Вопросы опроса, которые нужно задать: 
  
- Вы понимаете, как войти и использовать Поиск (Майкрософт)?
    
- Работает ли он так, как ожидалось?
    
- Какие функции вам нравятся?
    
- Удалось ли вам найти результаты работы?
    
- Полезны ли результаты работы?
    
- Понятны ли сообщения о Поиск (Майкрософт) и понятны?
    
Это отличное время для начала измерения эффективности показателей, выявленных на этапе 2.
  
При запуске пилотного этапа рассмотрите возможность совместного использования Поиск (Майкрософт) с более широкой организацией.
  
## <a name="stage-4-roll-out"></a>Этап 4. Раскатка

Каждая организация по-разному думает о выкатах продуктов. Вот несколько вещей, которые следует учитывать.
  
### <a name="broad-deployment"></a>Широкое развертывание
  
Когда вы будете готовы, либо разрешите всем пользователям организации доступ к Поиск (Майкрософт), либо используйте поэтапный подход, чтобы выкатиться медленнее. 
  
### <a name="generate-awareness"></a>Создание осведомленности
  
Создайте волнение в организации по поводу Поиск (Майкрософт):
  
- Привлечение пилотных пользователей для предоставления одноранговой помощи и экспертных знаний.
    
- Отслеживайте сообщения и события, запланированные на этапе 3.
    
### <a name="ongoing-communication"></a>Текущая связь
  
В течение первых нескольких недель и месяцев поддерживать ритм общения, чтобы стимулировать взаимодействие с Поиск (Майкрософт). Это позволяет пользователям быть в курсе и привлекать новых пользователей. Каждое сообщение должно объяснять, Поиск (Майкрософт) могут быть полезны для их работы.
  
Отправка текущих сообщений, которые выделяют функции и поиск, чтобы попробовать, а также истории успеха пользователей.
  
### <a name="gather-feedback-and-improve"></a>Сбор отзывов и улучшение
  
Поддержить пользователей и дать им знать, что вы заботитесь о них, слушая их и действуя на основе их отзывов.
  
Используйте панель мониторинга анализа, чтобы узнать, как выполняется продукт и содержимое. Используйте эту информацию для обновления закладки, Q&As и других сообщений. Это также поможет вам:
  
- Улучшение качества контента: просмотр названий, описаний и URL-адресов, оптимизация ключевых слов, удаление устаревших сведений и добавление новых сведений.
    
2. Определите, какие функции используются чаще или реже. 
    
3. Проводите опросы, фокус-группы и сеансы отзывов, чтобы понять, что пользователи думают о Поиск (Майкрософт), увеличении принятия и определении областей улучшения.
    
Образование и обратная связь — это итеративный процесс. Может потребоваться внести изменения в контент и сообщения по мере принятия пользователями Поиск (Майкрософт).
